Inspect the history of the repository 01

The middle column shows the list of commits that make up the history of the project. In the previous step, we left the repo like this:

Notice how there are two labels (leftmost in the middle column) with the text "master".

The name "master" is the name of the curent "branch" you're working on. Often, especially when working in teams, it is useful to separate changes from different people (or belonging to different features the team is working on) into separate sets of commits. Such a set is called a "branch", and "master" is the default name that Git uses for the first branch in the repo.

The top label has an icon referring to your local computer. The lower one has an icon referring to Github. This tells you that the repo on Github is two commits "behind" your local system. When you commit changes, all that happens is that the versions-database of your local repository changes. Github is left alone until you "push" the changes to Github (see the next page).
